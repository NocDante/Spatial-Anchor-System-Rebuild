# Spatial Anchor System Rebuild

---

## 📌 專案說明

這個專案的目的，是**重新寫一套舊專案中的 Spatial Anchor 系統**，同時作為一次重新整理觀念與學習的過程。

過去的開發主要依賴 Meta（Oculus）提供的 Unity 工具，功能可以用，但對底層運作其實不夠理解。隨著 Unity 與 Meta 的開發方式更新，本專案選擇**回到比較通用、比較基礎的做法**，重新理解 XR 專案在 Unity 中是怎麼運作的。

在這個過程中：
- 仍然會使用 Meta 提供的功能  
- 但不再只「照範例接 API」  
- 而是重新設計整個系統，理解每一步在做什麼  

這個專案同時也是：
- 讓自己重新熟悉 Unity 遊戲引擎的流程  
- 練習把程式寫得更清楚、更好維護  
- 建立正確的開發習慣，而不是只求功能完成  

---

## 🎯 專案目標

- 重寫舊有的 Spatial Anchor 系統，整理設計思路  
- 降低對特定工具的依賴，讓系統本身更好理解  
- 熟悉 Unity 在 XR 專案中的整體運作方式  
- 練習把程式責任分清楚，避免所有邏輯混在一起  

---

## 🧠 學習重點

- Spatial Anchor 在專案中實際扮演的角色  
- Anchor 的建立、保存與載入流程  
- Unity 專案中系統設計與程式結構的基本原則  
- 從「能動」到「寫得對、寫得久」的轉換
